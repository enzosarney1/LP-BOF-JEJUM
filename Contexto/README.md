# Workspace — Enzo Sarney

Este é o workspace compartilhado entre Enzo e Claude. Aqui ficam organizados todos os arquivos de contexto, projetos ativos, inputs e entregáveis.

---

## Estrutura de Pastas

### 📁 Contexto
Arquivos de referência permanente. Claude deve consultar esta pasta antes de qualquer trabalho — especialmente para produção de conteúdo, comunicação ou decisões estratégicas.

| Arquivo | O que contém |
|---|---|
| `BetterBE.md` | Tudo sobre a empresa: história, Golden Circle, produtos, ciência, modelo de negócio, público-alvo, aprendizados estratégicos |
| `Voz-da-Marca.md` | Como a BetterBE se comunica: tom de voz, arquétipos, vocabulário, exemplos reais de copy do Instagram e site |
| `Voz-Pessoal.md` | Como Enzo se comunica: estilo de escrita, guia para LinkedIn, o que evitar, checklist de revisão |
| `Sobre-Mim.md` | Quem é Enzo: dados pessoais, experiência, objetivos, identidade, caráter e visão de vida |
| `Preferencias-de-Trabalho.md` | Como trabalhamos juntos: fluxo de projetos, onde salvamos, formatos, regras gerais |

---

### 📁 Input
Arquivos enviados por Enzo como insumo para trabalhos. Claude lê daqui para executar tarefas. Nada é entregue a partir desta pasta — ela é só de entrada.

---

### 📁 Projetos
Arquivos de trabalho de projetos ativos em andamento. Cada projeto pode ter sua própria subpasta. Alguns projetos também serão gerenciados no ClickUp — Enzo indicará quando isso se aplicar.

---

### 📁 Output
Todos os entregáveis finais produzidos por Claude ficam aqui. É o destino padrão de qualquer arquivo gerado — documentos, análises, scripts, apresentações, etc.

---

### 📄 _Log.md
Registro histórico de tudo que foi produzido: data, projeto, arquivos entregues e status. Atualizado a cada entrega.

---

## Fluxo padrão de trabalho

1. Enzo envia briefing no chat
2. Claude consulta a pasta `Contexto/` para alinhar voz, perfil e objetivos
3. Claude apresenta opções de abordagem antes de executar
4. Enzo escolhe a direção
5. Claude executa e salva o entregável em `Output/`
6. Claude registra a entrega no `_Log.md`
